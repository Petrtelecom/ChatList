# ChatList

Программа для отправки одного промта в несколько нейросетей и сравнения их ответов.

## Возможности

- Отправка промта в несколько нейросетей одновременно
- Сравнение ответов от разных моделей в удобной таблице
- Сохранение промтов и результатов в базу данных
- Работа через OpenRouter API - один ключ для всех моделей
- Поддержка моделей от OpenAI, Anthropic, DeepSeek, Google, Meta и других провайдеров
- Выбор моделей для отправки запросов
- Параллельная отправка запросов для ускорения работы

## Требования

- Python 3.11+
- PyQt5
- requests
- python-dotenv

## Установка

1. Установите зависимости:
```powershell
pip install -r requirements.txt
```

2. Создайте файл `.env` на основе `.env.example`:
```powershell
Copy-Item .env.example .env
```

3. Получите API-ключ на [OpenRouter.ai](https://openrouter.ai) и заполните `.env` файл:
```
OPENROUTER_API_KEY=sk-or-v1-your-openrouter-api-key-here
```

**Важно:** Для работы программы нужен только один API-ключ от OpenRouter, который дает доступ ко всем моделям!

## Запуск

```powershell
python main.py
```

При первом запуске автоматически создастся база данных `chatlist.db` с предустановленными моделями через OpenRouter:
- GPT-4, GPT-3.5 Turbo (OpenAI)
- Claude 3.5 Sonnet, Claude 3 Opus (Anthropic)
- DeepSeek Chat
- Gemini Pro (Google)
- Llama 3 70B (Meta)

Все модели используют единый API-ключ OpenRouter.

## Использование

1. Введите промт в текстовое поле
2. Выберите модели, в которые нужно отправить запрос (чекбоксы)
3. Нажмите "Отправить запрос"
4. Дождитесь получения всех ответов
5. Выберите нужные результаты (чекбоксы) и нажмите "Сохранить выбранные"

## Структура проекта

- `main.py` - главное окно приложения (GUI)
- `db.py` - модуль работы с базой данных SQLite
- `models.py` - управление моделями нейросетей
- `network.py` - отправка запросов к API через OpenRouter
- `config.py` - конфигурационные настройки
- `DATABASE.md` - схема базы данных
- `PLAN.md` - план реализации программы

## База данных

База данных SQLite (`chatlist.db`) содержит:
- **prompts** - сохраненные промты
- **models** - список моделей нейросетей (все через OpenRouter)
- **results** - сохраненные результаты ответов
- **settings** - настройки приложения

Подробная схема описана в файле `DATABASE.md`.

## Примечание

Если у вас уже была создана база данных со старыми моделями, удалите файл `chatlist.db` перед запуском, чтобы создать новую БД с моделями OpenRouter.
